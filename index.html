Trabalho do Professor Matheus Ferraz

Ao longo das aulas do professor Matheus Ferraz, foram abordados diversos conceitos e comandos relacionados do Git.

Configuração do Git:

Utilizar git config -l para listar as configurações do Git.

Comandos Básicos:

git init (Inicialização do Repositório): Utilizado para iniciar um novo repositório Git em um diretório existente. git status: Verificar o status do repositório. git add : Adicionar um arquivo específico ao staged. git add *.txt: Adicionar todos os arquivos com extensão .txt ao staged. git commit -m "Mensagem": Realizar um commit com uma mensagem. git log: Exibir o histórico de commits.

Manipulação de Arquivos:

git rm : Remover um arquivo do repositório. git rm --cached : Remover um arquivo do staged sem excluí-lo localmente. git rm -rf : Remover uma pasta do staged e do repositório.

Ignorando Arquivos:

Criar e usar um arquivo .gitignore para listar os arquivos a serem ignorados.

Revertendo Alterações:

git reset: Desfazer o último commit sem perder as alterações locais. git revert: Desfazer um commit criando um novo commit que reverte as alterações.

Branches:

git branch: Listar todas as branches no repositório. git checkout -b : Criar e mudar para uma nova branch. git merge : Realizar a fusão de branches. git branch -D : Deletar uma branch.

Stash:

git stash: Guardar temporariamente as alterações locais. git stash pop: Aplicar as últimas alterações guardadas. git stash list: Listar todas as alterações guardadas.

GitHub:

Configurar um repositório remoto no GitHub. Gerar um token pessoal para autenticação no GitHub.

GitHub Pages:

Configurar manual ou automaticamente (GitHub Actions) para publicar uma página no GitHub Pages.

GitHub Actions:

Utilizar GitHub Actions para automação de tarefas no fluxo de trabalho do Git.

GitHub Settings:

Explorar configurações e opções no GitHub, incluindo personal access tokens e audit log.

Visualização Avançada de Logs:

Utilizar diferentes opções do git log para visualizar informações específicas dos commits.

Revisão e Revertimento:

Utilizar comandos como git diff --staged, git commit --amend, git revert, e git reset para revisar e reverter alterações.

GitHub Organization:

Criar uma nova organização no GitHub para gerenciar repositórios.

Outros Conceitos:

Conceitos adicionais como arquivos ocultos, .gitkeep, e HEAD. .gitignore (Ignorar Arquivos) Arquivo usado para especificar quais arquivos e diretórios o Git deve ignorar durante o versionamento. git clone (Clonar Repositório) Usado para criar uma cópia local de um repositório Git existente.
